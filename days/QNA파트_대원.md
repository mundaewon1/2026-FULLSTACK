# 1. 역할 분담

## 프론트엔드

### 담당 기능
* 사용자 ↔ 관리자 1:1 문의 화면 구현
* 사용자 ↔ 모임글 게시자 1:1 문의 화면 구현
* 문의 작성, 조회, 수정, 삭제 기능 구현
* 답변 조회 화면 구현
* 문의 관련 UI 설계 및 화면 구성

## 백엔드

### 담당 기능
* 게시자 문의 접수 기능 구현
* 관리자 문의 접수 기능 구현
* 문의,답변의 등록, 조회, 수정, 삭제 기능 구현
* 문의 상태(답변 대기, 답변 완료) 관리 기능 구현
* 문의 데이터 저장 및 관리

# 2. 자체 평가

## 긍정적인 부분

### 잘 진행된 점
* 사용자와 관리자, 게시자 간 1:1 문의 기능을 구현
* 문의 상태 관리 기능과 통계 기능을 구현
* 답변 수, 오늘 등록 수, 답변 대기 수 등의 통계 기능을 구현
* 프론트엔드와 백엔드를 연동하여 문의 및 답변 등록 기능을 사용할 수 있도록 구현

### 프로젝트에 기여한 부분
* 1:1 문의 기능의 화면 구현 및 기능 개발을 담당
* 문의 데이터 처리 기능 구현에 참여
* 답변 수, 오늘 등록 수, 답변 대기 수 등의 통계 기능 구현에 참여하였습니다.
* 기능 테스트 및 오류 수정에 참여하여 프로젝트 완성에 기여하였습니다.

## 아쉬웠던 부분

### 구현하지 못한 기능
* 문의,답변의 조회, 수정, 삭제 기능 테스트는 마쳤으나, 화면 연동은 완료하지 못하였습니다.

### 일정 및 협업 과정에서 부족했던 점
* 첫 팀 프로젝트라 기능 구현 과정에서 예상보다 많은 시간이 소요되었습니다.
* 다양한 사용자 환경에서 충분한 테스트를 진행하지 못하였습니다.

## 개선(발전) 사항

### 추후 보완할 기능
* 문의,답변의 조회, 수정, 삭제 기능 화면 연동
* api 추가

### 성능 및 UI/UX 개선 방향
* 문의 내역 화면의 가독성 개선
* 사용자 편의성을 고려한 UI 개선
* 문의 작성 및 조회 화면 디자인 개선

### 추가 구현 아이디어 (여유되면)
* 실시간 채팅 기능
* 문의 알림 기능
* 자주 묻는 질문(FAQ) 기능
